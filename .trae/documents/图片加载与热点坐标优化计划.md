# 图片加载与热点坐标优化计划

## 一、问题综合分析

经过对项目代码的深入阅读和三个AI review意见的融合，我将核心问题归纳为以下三大类：

---

### 🔴 问题A：图片加载与显示异常（最严重）

**用户表现**：往后翻页时显示前一张图片；网络差时图片闪烁/空白。

**根因分析**（融合三方意见 + 自身验证）：

1. **`waitForImageLoad` 的致命缺陷**（三方共识，最核心问题）
   - 当 `<img>` 元素之前已经加载过图片时，`imgEl.complete` 为 `true`，`naturalWidth > 0`
   - 设置新的 `src` 后，浏览器开始异步加载新图片，但 `complete` 属性可能**短暂保持 `true`**（旧图片状态残留）
   - `waitForImageLoad` 检查 `imgEl.complete && imgEl.naturalWidth > 0` 时立即返回，实际新图片还没加载完
   - **结果**：CSS 类立即切换，但新图片未就绪 → 显示旧图片或空白

2. **预加载缓存未被真正利用**（三方共识）
   - `state.preloadedImages[src]` 存储了 `Image` 对象
   - 但 `renderScene` 中直接 `nextImg.src = scene.image`，让浏览器重新请求
   - 虽然浏览器 HTTP 缓存可能命中，但在无本地缓存/网络差时，会重新下载

3. **预加载失败无重试**（三方共识）
   - `img.onerror` 中直接 `resolve()`，失败后不重试
   - 后续渲染时再次加载，但网络可能更差

4. **初始化阻塞用户体验**（两个AI提到）
   - `await preloadAllImages()` 必须等所有图片加载完
   - 网络差时长时间黑屏，无进度反馈

5. **`isTransitioning` 锁定时间不精确**（三方共识）
   - 固定 1000ms 解锁，但图片加载可能超过 1s
   - 用户可能在图片未完全显示时再次点击

---

### 🔴 问题B：热点坐标偏移

**用户表现**：热点位置与图片上产品实际位置不一致。

**根因分析**：

1. **热点计算依赖 `naturalWidth/naturalHeight`，但图片可能未加载完成**（三方共识）
   - `naturalWidth || 1` 的回退值导致计算完全错误
   - 图片加载中时 `naturalWidth` 可能为 0 或旧图片的尺寸

2. **手动计算 `object-fit: cover` 裁切偏移**（我的分析 + 一个AI提到）
   - 当前代码手动推算 cover 模式的渲染区域和裁切偏移
   - 这个计算本身逻辑是正确的，**但前提是 `naturalWidth/naturalHeight` 必须准确**
   - 更好的方案：使用 `getBoundingClientRect()` 获取实际渲染尺寸，避免手动计算可能的细微误差

3. **热点渲染时机与图片加载不同步**（三方共识）
   - 热点在 `setTimeout(() => renderHotspot(), 800)` 后渲染
   - 如果图片加载超过 800ms，热点计算时图片可能未完全渲染
   - 如果图片加载很快，800ms 的等待又是不必要的延迟

4. **`activeLayer` 标记与 CSS 类切换顺序问题**（一个AI提到）
   - 先切换 CSS 类，后切换 `state.activeLayer`
   - 如果在 800ms 等待期间触发 `repositionHotspot()`，会使用错误的图层计算

5. **resize 时无防抖且不检查图片加载状态**（两个AI提到）
   - 频繁 resize 导致频繁计算
   - 切换过程中 resize 可能拿到错误尺寸

---

### 🟡 问题C：缺少加载状态反馈

1. 无 loading 指示器，用户不知道图片是否在加载
2. 无加载超时机制，图片加载卡住时场景切换会一直等待
3. 无加载失败降级处理（占位图/错误提示）

---

## 二、优化方案（最终融合版）

### 核心原则
- **保持简单**：不引入复杂框架，纯原生 JS
- **渐进式修改**：每个步骤独立可测试
- **详细注释**：每处修改添加中文注释

---

### 优化1：修复 `waitForImageLoad` — 真正等待新图片加载

**方案**：在设置新 `src` 之前，先将 `<img>` 元素重置，确保 `complete` 变为 `false`。

```javascript
function waitForImageLoad(imgEl, timeoutMs = 8000) {
    return new Promise((resolve) => {
        let resolved = false;
        let timeoutId = null;

        const done = () => {
            if (resolved) return;
            resolved = true;
            if (timeoutId) clearTimeout(timeoutId);
            resolve();
        };

        // 如果图片已经加载完成且有效
        if (imgEl.complete && imgEl.naturalWidth > 0) {
            resolve();
            return;
        }

        // 监听加载完成和错误
        imgEl.addEventListener('load', done, { once: true });
        imgEl.addEventListener('error', done, { once: true });

        // 超时保护
        timeoutId = setTimeout(done, timeoutMs);
    });
}
```

**关键改进**：
- 使用 `{ once: true }` 避免内存泄漏
- 添加 8 秒超时保护
- 使用 `addEventListener` 替代 `onload` 赋值，避免覆盖其他监听器

**但最关键的改进在调用侧**：在设置 `src` 之前先清除，确保 `complete` 为 `false`：

```javascript
// 在 renderScene 中，设置新图片前先重置
nextImg.removeAttribute('src');  // 清除旧src，确保complete变为false
nextImg.src = scene.image;       // 设置新src，触发加载
await waitForImageLoad(nextImg); // 现在能正确等待新图片
```

### 优化2：改进初始化流程 — 非阻塞 + 优先首屏

**方案**：不再等待所有图片预加载，立即显示第一个场景。

```javascript
async function init() {
    initSceneCategories();
    createSwitcher();
    createIndicator();

    // 立即加载并显示第一个场景（不等待所有预加载）
    const scene = scenes[0];
    dom.sceneImageA.removeAttribute('src');
    dom.sceneImageA.src = scene.image;
    dom.sceneImageA.alt = scene.name;

    await waitForImageLoad(dom.sceneImageA);

    dom.sceneImageA.classList.add('active');
    dom.sceneImageA.classList.remove('inactive');

    updateIndicator(0);
    updateSwitcher(scene.name);
    renderHotspot(scene.hotspot);
    dom.sceneSwitcher.classList.add('visible');

    // 后台异步预加载其他图片（不阻塞首屏显示）
    preloadAllImages().then(count => {
        console.log(`后台预加载完成：${count}张`);
    });

    bindEvents();
    addHintText();
}
```

### 优化3：改进预加载 — 添加重试机制

```javascript
const preloadOne = (src, retries = 2) => {
    return new Promise((resolve) => {
        if (state.preloadedImages[src]) {
            resolve();
            return;
        }

        const tryLoad = (attempt) => {
            const img = new Image();
            img.onload = () => {
                state.preloadedImages[src] = img;
                resolve();
            };
            img.onerror = () => {
                if (attempt < retries) {
                    // 递增延迟重试
                    setTimeout(() => tryLoad(attempt + 1), 500 * (attempt + 1));
                } else {
                    console.warn(`画像のプリロードに失敗（${retries + 1}回試行）: ${src}`);
                    resolve();
                }
            };
            img.src = src;
        };
        tryLoad(0);
    });
};
```

### 优化4：改进 `renderScene` — 修复竞态条件 + 精确锁定

**核心改动**：
1. 设置 `src` 前先清除，确保 `waitForImageLoad` 能正确等待
2. `isTransitioning` 在图片加载完成 + CSS 过渡完成后才解锁
3. 热点在图片加载完成后立即渲染，不用 `setTimeout 800ms`
4. 先切换 `activeLayer` 标记，再切换 CSS 类

```javascript
async function renderScene(index, animate = true) {
    const scene = scenes[index];
    if (!scene) return;

    if (animate) {
        const currentImg = state.activeLayer === 'a' ? dom.sceneImageA : dom.sceneImageB;
        const nextImg = state.activeLayer === 'a' ? dom.sceneImageB : dom.sceneImageA;

        // 隐藏热点和切换器
        dom.hotspotContainer.style.opacity = '0';
        dom.sceneSwitcher.style.opacity = '0';

        // ★ 关键：先清除src，确保complete变为false
        nextImg.removeAttribute('src');
        // 设置新图片
        nextImg.src = scene.image;
        nextImg.alt = scene.name;

        // ★ 等待新图片真正加载完成（带超时保护）
        await waitForImageLoad(nextImg);

        // ★ 先切换activeLayer标记，确保后续repositionHotspot使用正确图层
        state.activeLayer = state.activeLayer === 'a' ? 'b' : 'a';

        // 执行交叉淡入淡出
        nextImg.classList.add('active');
        nextImg.classList.remove('inactive');
        currentImg.classList.remove('active');
        currentImg.classList.add('inactive');

        // 更新UI
        updateIndicator(index);
        updateSwitcher(scene.name);

        // ★ 图片加载完成后立即渲染热点（不再用setTimeout 800ms）
        // 使用requestAnimationFrame确保浏览器已完成一帧渲染
        requestAnimationFrame(() => {
            renderHotspot(scene.hotspot);
            dom.hotspotContainer.style.opacity = '1';
            dom.sceneSwitcher.style.opacity = '1';
        });
    } else {
        // 无动画模式逻辑类似...
    }
}
```

**`prevScene/nextScene/goToScene` 的锁定改进**：

```javascript
function prevScene() {
    if (state.isTransitioning || state.isDetailOpen) return;
    state.isTransitioning = true;
    state.currentIndex = (state.currentIndex - 1 + scenes.length) % scenes.length;
    renderScene(state.currentIndex).then(() => {
        // ★ 等CSS过渡完成后再解锁（transition 1.2s + 余量）
        setTimeout(() => { state.isTransitioning = false; }, 1300);
    });
}
```

### 优化5：改进热点坐标计算 — 使用 `getBoundingClientRect`

**方案**：不再手动计算 `object-fit: cover` 的裁切偏移，改用浏览器实际渲染尺寸。

**分析**：当前代码手动计算 cover 裁切偏移的逻辑是正确的，但存在两个问题：
1. 依赖 `naturalWidth/naturalHeight`，图片未加载时值不准确
2. 手动计算可能与浏览器实际渲染有细微差异

**改进方案**：使用 `getBoundingClientRect()` 获取图片元素的实际渲染区域。

**但有一个关键问题**：`object-fit: cover` 模式下，`<img>` 元素的 `getBoundingClientRect()` 返回的是**容器尺寸**（100% x 100%），不是图片实际渲染区域。图片可能被裁切，裁切部分超出了容器。

**因此正确的方案是**：保留手动计算 cover 裁切偏移的逻辑，但增加图片加载状态检查，确保 `naturalWidth/naturalHeight` 准确。

```javascript
function calcHotspotPixelPosition(position) {
    const activeImg = state.activeLayer === 'a' ? dom.sceneImageA : dom.sceneImageB;

    // ★ 关键：确保图片已完全加载
    if (!activeImg.complete || activeImg.naturalWidth === 0) {
        console.warn('画像未読み込み、ホットスポット計算スキップ');
        // 返回屏幕中心作为临时位置
        return { x: window.innerWidth / 2, y: window.innerHeight / 2 };
    }

    const containerW = window.innerWidth;
    const containerH = window.innerHeight;
    const imgW = activeImg.naturalWidth;
    const imgH = activeImg.naturalHeight;

    const imgAspect = imgW / imgH;
    const containerAspect = containerW / containerH;

    let renderedW, renderedH, cropOffsetX, cropOffsetY;

    if (imgAspect > containerAspect) {
        renderedH = containerH;
        renderedW = containerH * imgAspect;
        cropOffsetX = (renderedW - containerW) / 2;
        cropOffsetY = 0;
    } else {
        renderedW = containerW;
        renderedH = containerW / imgAspect;
        cropOffsetX = 0;
        cropOffsetY = (renderedH - containerH) / 2;
    }

    return {
        x: (position.x / 100) * renderedW - cropOffsetX,
        y: (position.y / 100) * renderedH - cropOffsetY
    };
}
```

### 优化6：改进 `repositionHotspot` — 添加防抖和安全检查

```javascript
let resizeTimer = null;

window.addEventListener('resize', () => {
    // 防抖：200ms内只执行一次
    clearTimeout(resizeTimer);
    resizeTimer = setTimeout(() => {
        repositionHotspot();
    }, 200);
});
```

```javascript
function repositionHotspot() {
    const scene = scenes[state.currentIndex];
    if (!scene) return;

    const hotspot = dom.hotspotContainer.querySelector('.hotspot');
    if (!hotspot) return;

    // ★ 确保图片已加载再计算
    const activeImg = state.activeLayer === 'a' ? dom.sceneImageA : dom.sceneImageB;
    if (!activeImg.complete || activeImg.naturalWidth === 0) return;

    const pixelPos = calcHotspotPixelPosition(scene.hotspot);
    hotspot.style.left = pixelPos.x + 'px';
    hotspot.style.top = pixelPos.y + 'px';
}
```

## 五、关于热点坐标计算方案的补充说明

三个AI中有一个建议用 `getBoundingClientRect()` 替代手动计算 cover 裁切偏移。经过我的分析：

**`getBoundingClientRect()` 在 `object-fit: cover` 模式下不适用**，原因是：
- `<img>` 元素设置了 `width: 100%; height: 100%`，所以 `getBoundingClientRect()` 返回的是容器的全屏尺寸
- 但图片实际渲染区域可能超出容器（被裁切的部分），`getBoundingClientRect()` 无法反映这个信息
- 简单地用 `rect.left + (position.x / 100) * rect.width` 计算，等价于把热点位置映射到容器上，**没有考虑 cover 裁切**

**结论**：保留当前的手动计算 cover 裁切偏移方案，但增加图片加载状态检查，确保 `naturalWidth/naturalHeight` 准确。这是最可靠的方案。

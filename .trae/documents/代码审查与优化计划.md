## 一、发现的问题清单

### 🔴 高优先级问题

#### 问题1：初始化流程阻塞用户体验
**位置**：`js/main.js` 第860行
**问题描述**：
```javascript
await preloadAllImages();  // 必须等所有图片加载完才继续
```
- 在网络条件差时，用户会长时间看到黑屏
- 即使只有一张图片加载失败，也要等所有图片都尝试加载完
- 没有加载进度反馈

#### 问题2：快速切换场景时图片显示混乱
**位置**：`js/main.js` 第398-419行
**问题描述**：
- 双层交叉淡入淡出时，没有对图片加载进行排队管理
- 如果用户快速点击切换，新图片设置后，旧图片的onload事件可能还在处理
- 可能导致显示错误的图片（下一张显示前一张）

#### 问题3：热点坐标计算与图层切换时机不同步
**位置**：`js/main.js` 第418-429行
**问题描述**：
```javascript
// 先切换CSS类
currentImg.classList.remove('active');
currentImg.classList.add('inactive');

// 后切换activeLayer标记（第421行）
state.activeLayer = state.activeLayer === 'a' ? 'b' : 'a';

// 800ms后才渲染热点
setTimeout(() => {
    renderHotspot(scene.hotspot);
    // ...
}, 800);
```
- 如果在这800ms内窗口大小改变，`repositionHotspot()` 会使用错误的图层计算坐标
- 热点可能出现在错误位置

#### 问题4：isTransitioning锁解除时机不可靠
**位置**：`js/main.js` 第459行
**问题描述**：
```javascript
setTimeout(() => { state.isTransitioning = false; }, 1000);
```
- 用固定的1000ms解除锁，而不是等图片真正加载完成
- 如果网络慢，图片加载超过1000ms，用户可以继续点击切换，导致多个切换同时进行

---

### 🟡 中优先级问题

#### 问题5：图片加载没有超时和降级处理
**位置**：`js/main.js` 第316-324行
**问题描述**：
- `waitForImageLoad()` 只有onload和onerror，没有超时机制
- 图片加载卡住时，场景切换会一直等待
- 没有loading占位图或失败重试

#### 问题6：预加载没有本地持久化
**位置**：`js/main.js` 第246-299行
**问题描述**：
- 每次刷新页面都要重新预加载所有图片
- 没有利用LocalStorage记录已加载的图片
- 没有利用HTTP缓存的最佳实践

#### 问题7：热点位置计算时缺少图片尺寸检查
**位置**：`js/main.js` 第590-623行
**问题描述**：
```javascript
const imgW = activeImg.naturalWidth || 1;  // 用1作为默认值太随意
const imgH = activeImg.naturalHeight || 1;
```
- 如果图片还没加载完成，`naturalWidth` 为0
- 会导致热点计算完全错误

## 二、优化方案详细设计

### 优化方案1：改进初始化流程（非阻塞+进度提示）

**目标**：让用户尽快看到第一个场景，其他图片后台加载

**修改内容**：
1. 删除 `init()` 中的 `await preloadAllImages()`
2. 立即加载并显示第一个场景
3. 后台异步预加载其他图片
4. 添加加载进度提示（可选）

**代码结构**：
```javascript
async function init() {
    // 1. 立即初始化UI元素
    initSceneCategories();
    createSwitcher();
    createIndicator();
    
    // 2. 立即加载并显示第一个场景（不等待预加载）
    const firstScene = scenes[0];
    dom.sceneImageA.src = firstScene.image;
    await waitForImageLoad(dom.sceneImageA);  // 只等第一个场景
    
    // 3. 显示第一个场景
    dom.sceneImageA.classList.add('active');
    dom.sceneImageA.classList.remove('inactive');
    updateIndicator(0);
    updateSwitcher(firstScene.name);
    renderHotspot(firstScene.hotspot);
    dom.sceneSwitcher.classList.add('visible');
    
    // 4. 后台异步预加载其他图片（不阻塞）
    preloadAllImages().then(count => {
        console.log(`后台预加载完成：${count}/${imagePaths.size}张`);
    });
    
    // 5. 绑定事件
    bindEvents();
    addHintText();
}
```

### 优化方案3：热点计算与图层切换同步

**目标**：确保热点始终基于正确的图片计算

**修改点**：
1. 先切换 `activeLayer` 标记，再切换CSS类
2. 热点渲染不要等800ms，图片加载完成后立即渲染
3. 在 `repositionHotspot()` 中检查热点是否存在
4. 在 `calcHotspotPixelPosition()` 中确保图片已加载完成

**修改后的顺序**：
```javascript

// 2. 再切换CSS类
nextImg.classList.add('active');
nextImg.classList.remove('inactive');
currentImg.classList.remove('active');
currentImg.classList.add('inactive');

// 3. 立即渲染热点（不要等800ms）

```

### 优化方案4：添加图片加载超时和降级处理

**改进 `waitForImageLoad()` 函数**：
```javascript
function waitForImageLoad(imgEl, timeoutMs = 5000) {
    return new Promise((resolve) => {
        // 1. 如果已经加载完成，立即resolve
        if (imgEl.complete && imgEl.naturalWidth > 0) {
            resolve();
            return;
        }
        
        let timeoutId = null;
        let resolved = false;
        
        const done = () => {
            if (resolved) return;
            resolved = true;
            if (timeoutId) clearTimeout(timeoutId);
            resolve();
        };
        
        // 2. onload
        imgEl.onload = done;
        
        // 3. onerror
        imgEl.onerror = done;
        
        // 4. 超时机制
        timeoutId = setTimeout(done, timeoutMs);
    });
}
```

### 优化方案6：热点渲染增加安全检查

**改进 `repositionHotspot()` 函数**：
```javascript
function repositionHotspot() {
    const scene = scenes[state.currentIndex];
    if (!scene) return;
    
    // 1. 检查热点是否存在
    const hotspot = dom.hotspotContainer.querySelector('.hotspot');
    if (!hotspot) {
        // 如果不存在，重新渲染
        renderHotspot(scene.hotspot);
        return;
    }
    
    // 2. 正常重新定位
    const pixelPos = calcHotspotPixelPosition(scene.hotspot);
    hotspot.style.left = pixelPos.x + 'px';
    hotspot.style.top = pixelPos.y + 'px';
}
```

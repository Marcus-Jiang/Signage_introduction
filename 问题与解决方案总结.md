# デジタルサイネージ製品紹介ページ - 問題と解決策まとめ

## プロジェクト概要

10シーンのデジタルサイネージ製品紹介ページ。各シーンにホットスポットがあり、クリックで製品詳細パネルが表示される。2層`<img>`要素によるクロスフェード、`object-fit: cover`による全画面画像表示、パーセント座標によるホットスポット位置指定などが特徴。

---

## 問題1：ページ切替時に前の画像が表示される

### 現象
次のシーンに切り替えると、新しい画像ではなく前の画像が一瞬表示される、または新しい画像が全く表示されない。

### 根因
`waitForImageLoad` 関数の致命的な欠陥：

```
<img>要素が既に画像を読み込み済みの場合：
  img.complete = true
  img.naturalWidth > 0

新しいsrcを設定した直後：
  ブラウザは非同期で新しい画像のダウンロードを開始
  しかし img.complete は一時的に true のまま（旧画像の状態が残留）

waitForImageLoad のチェック：
  if (imgEl.complete && imgEl.naturalWidth > 0) → 即座に resolve()
  → 新しい画像のダウンロード完了を待っていない！
```

### 解決策
`<img>`要素の`src`を設定する前に、`removeAttribute('src')`でクリアする：

```javascript
// 修正前
nextImg.src = scene.image;
await waitForImageLoad(nextImg); // 旧画像のcomplete=trueで即座に解決

// 修正後
nextImg.removeAttribute('src');  // complete を false にリセット
nextImg.src = scene.image;
await waitForImageLoad(nextImg); // 新しい画像のダウンロード完了を正しく待機
```

## 問題2：ホットスポットの位置がずれる

### 現象
ホットスポット（点滅する丸いマーカー）が、画像上の製品の実際の位置と一致しない。

### 根因
複合的な原因：

1. **`naturalWidth || 1` の危険なフォールバック**
   - 画像未ロード時、`naturalWidth`が0 → `|| 1`で1に置換
   - アスペクト比計算が完全に破綻（1/画像高さ = 極端な値）

2. **ホットスポットのレンダリングが画像ロードと非同期**
   - `setTimeout(() => renderHotspot(), 800)` で800ms後にホットスポット描画
   - 画像ロードが800ms以上かかる場合、未ロードの画像サイズで計算
   - 画像ロードが800ms未満の場合、不要な800ms待ちが発生

3. **`activeLayer`マークとCSS切替の順序問題**
   - CSS切替後に`activeLayer`を更新 → 切替中に`repositionHotspot`が誤ったレイヤを参照

### 解決策

```javascript
// 1. naturalWidth || 1 を廃止、画像ロード完了チェックを追加
if (!activeImg.complete || activeImg.naturalWidth === 0) {
    return { x: containerW / 2, y: containerH / 2 }; // 安全な仮位置
}

// 2. setTimeout(800ms) → requestAnimationFrame に変更
// 画像ロード完了直後にホットスポットを即時描画
requestAnimationFrame(() => {
    renderHotspot(scene.hotspot);
});

// 3. activeLayerマークをCSS切替の前に更新
state.activeLayer = state.activeLayer === 'a' ? 'b' : 'a'; // 先に更新
nextImg.classList.add('active');  // その後にCSS切替
```

## 問題3：黒画面に孤立したホットスポットが表示される

### 現象
ネットワークが遅い環境で、画像がまだ表示されていない黒い画面に、ホットスポットだけが浮いて表示される。ホットスポットの位置も間違っている。

### 根因
`waitForImageLoad`がタイムアウトまたはエラーで`resolve(false)`を返した後も、ホットスポットのレンダリングが実行されていた：

```javascript
// 修正前
await waitForImageLoad(nextImg); // タイムアウトしても処理が続行
// ...
renderHotspot(scene.hotspot);    // 画像未ロードでもホットスポット描画
```

### 解決策
`waitForImageLoad`の戻り値（`true`/`false`）をチェックし、画像ロード成功時のみホットスポットを描画：

```javascript
const loadSuccess = await waitForImageLoad(nextImg);

if (loadSuccess || actuallyLoaded) {
    // 画像が表示されている場合のみホットスポット描画
    requestAnimationFrame(() => {
        renderHotspot(scene.hotspot);
    });
} else {
    // ロード失敗時はホットスポットを描画しない
    console.warn('画像ロード失敗');
}
```

## 問題4：首画面の画像が永遠に表示されない（慢速4G環境で100%再現）

### 現象
慢速4Gまたはそれ以下のネットワーク環境でサイトを開くと、全画像のプリロードが完了しても、首画面のシーン画像が黒いまま永遠に表示されない。

### 根因
**二重の致命的組み合わせ**：

**要因1：`preloadAllImages()`と首画面画像の帯域争奪**

```
0ms   首画面画像 img.src = scene.image  → ダウンロード開始
0ms   preloadAllImages()                → 同時に20枚のダウンロード開始
      ↓
慢速4G：ブラウザの同時接続数上限6本
首画面画像は全帯域の1/6しか使えない
元々8秒で完了するダウンロードが48秒+に延長
```

**要因2：8秒タイムアウトが短すぎる**

```
0s      首画面画像ダウンロード開始（帯域1/6）
8s      waitForImageLoad タイムアウト → loadSuccess=false
        → 画像のCSSクラスが永遠にactiveに切り替わらない
30s+    首画面画像の実際のダウンロード完了
        → しかし resolved=true で onLoad は無視される
        → 画像データはDOMにあるが、永遠に非表示
60s+    preloadAllImages 完了
        → サイトは「ロード完了」だが、首画面は黒いまま
```

### 解決策

**三管斉下の修正**：

1. **`preloadAllImages()`を首画面画像ロード完了後に移動**
   ```javascript
   // 修正前：首画面画像と同時にプリロード開始
   img.src = scene.image;
   preloadAllImages(); // ← 帯域争奪！

   // 修正後：首画面画像が表示されてからプリロード開始
   img.src = scene.image;
   const loadSuccess = await waitForImageLoad(img, 30000);
   // 首画面画像が表示された後...
   preloadAllImages(); // ← 帯域争奪なし
   ```

2. **首画面画像のタイムアウトを8秒→30秒に延長**
   ```javascript
   // 慢速ネットワーク対応
   await waitForImageLoad(img, 30000); // 首画面は30秒
   await waitForImageLoad(nextImg, 15000); // シーン切替は15秒
   ```

3. **タイムアウト後の二次チェック（安全網）**
   ```javascript
   const loadSuccess = await waitForImageLoad(img, 30000);
   // タイムアウト直前に画像がロード完了した場合の安全網
   const actuallyLoaded = img.complete && img.naturalWidth > 0;
   if (loadSuccess || actuallyLoaded) {
       // 画像を表示
   }
   ```

## 問題5：ローディングインジケータ（回転アニメ）の挙動が不正

### 現象
- 首画面の画像ロード中に回転アニメが表示されない
- 画像が既にキャッシュされているのに、毎回シーン切替で回転アニメが表示される

### 根因

1. **首画面でローディング表示がない**：`init()`関数でローディングインジケータの表示処理が欠落していた

2. **キャッシュ済み画像でもローディング表示**：画像が`state.preloadedImages`にキャッシュされているかどうかをチェックせず、毎回ローディング表示していた

### 解決策

```javascript
// キャッシュチェック関数を追加
function isImageCached(src) {
    return !!state.preloadedImages[src];
}

// ローディング表示条件：キャッシュされていない場合のみ
const needLoading = !isImageCached(scene.image);
if (needLoading) {
    dom.loadingIndicator.classList.add('visible');
}

// ...画像ロード完了後...
if (needLoading) {
    dom.loadingIndicator.classList.remove('visible');
}
```

| シーン | needLoading | ローディング表示 |
|--------|-------------|-----------------|
| 首画面（初回） | 常にtrue | ★表示 |
| プリロード完了前のシーン切替 | true（未キャッシュ） | ★表示 |
| プリロード完了後のシーン切替 | false（キャッシュ済） | ✗非表示 |
| プリロード失敗のシーン | true（未キャッシュ） | ★表示 |

## 問題6：詳細パネルの製品説明が遅い

### 現象
ホットスポットをクリック後、製品の画像と説明テキストがなかなか表示されない。

### 根因
`renderProductList`で製品のMarkdown説明を**直列**読み込みしていた：

```javascript
// 修正前：直列読み込み（3製品×各1秒 = 3秒）
for (const product of products) {
    const markdown = await loadDescription(product.descriptionFile); // 待機
    descEl.innerHTML = parseMarkdown(markdown);
    dom.productList.appendChild(item);
}
```

### 解決策

1. **先にDOM骨組みを即座に表示**（ローディングプレースホルダ付き）
2. **Markdown説明を並列読み込み**に変更

```javascript
// 第一步：全製品のDOM骨組みを即座に作成
for (const product of products) {
    // 製品名と画像は即座に表示
    // 説明エリアは「読み込み中...」プレースホルダ
    descEl.innerHTML = '<div class="desc-loading">読み込み中...</div>';
}

// 第二步：全説明を並列読み込み（3製品×各1秒 = 1秒）
const loadPromises = descElements.map(async ({ el, file }) => {
    const markdown = await loadDescription(file);
    el.innerHTML = parseMarkdown(markdown);
});
await Promise.all(loadPromises);
```

## 問題7：ウィンドウリサイズ時にホットスポットがずれる

### 現象
ブラウザウィンドウのサイズを変更すると、ホットスポットの位置がずれる。頻繁にリサイズするとホットスポットが点滅する。

### 根因
1. リサイズイベントにデバウンスがない → 毎回即座に再計算
2. 画像未ロード時の再計算で不正確な値になる可能性

### 解決策

```javascript
// デバウンス付きリサイズイベント
let resizeTimer = null;
window.addEventListener('resize', () => {
    clearTimeout(resizeTimer);
    resizeTimer = setTimeout(() => {
        repositionHotspot();
    }, 200);
});

// repositionHotspot内で画像ロード状態チェック
function repositionHotspot() {
    const activeImg = state.activeLayer === 'a' ? dom.sceneImageA : dom.sceneImageB;
    if (!activeImg.complete || activeImg.naturalWidth === 0) return; // 安全チェック
    // ...
}
```

## その他の改善

### プリロードのリトライ機能

ネットワーク不安定な環境での画像ロード成功率を向上：

```javascript
const preloadOne = (src, retries = 2) => {
    const tryLoad = (attempt) => {
        const img = new Image();
        img.onload = () => { /* 成功 */ };
        img.onerror = () => {
            if (attempt < retries) {
                // リトライ：遅延递增（500ms、1000ms）
                setTimeout(() => tryLoad(attempt + 1), 500 * (attempt + 1));
            }
        };
        img.src = src;
    };
    tryLoad(0);
};
```

### `waitForImageLoad`の改善

| 項目 | 修正前 | 修正後 |
|------|--------|--------|
| イベントリスナー | `img.onload =`（上書きの危険） | `addEventListener` + `{ once: true }` |
| メモリリーク | リスナーの削除なし | `{ once: true }`で自動削除 |
| タイムアウト | なし（永遠に待機の危険） | デフォルト8秒、首画面30秒 |
| 戻り値 | `void`（成功/失敗不明） | `boolean`（true=成功、false=失敗） |

## 修正版のローディングフロー全体図

```
ページを開く
  │
  ├─ 1. UI要素を作成（タブ、インジケータ）
  ├─ 2. ★ローディング表示（回転アニメ）
  ├─ 3. 首画面画像のダウンロード開始（全帯域を独占）
  │
  ├─ ... ダウンロード中（回転アニメ表示中）...
  │
  ├─ 4. 首画面画像ダウンロード完了
  │   ├─ ローディング非表示
  │   ├─ 画像フェードイン（CSS transition 1.2s）
  │   ├─ ホットスポット即時描画
  │   └─ ★プリロード開始（ここから帯域を共有）
  │
  ├─ ... バックグラウンドで残り19枚をプリロード...
  │
  ├─ 5. プリロード完了
  │   └─ 以降のシーン切替はキャッシュから瞬時に表示
  │      （ローディングアニメなし）
  │
  └─ ユーザー操作
      ├─ 左右ボタン/キーボード → シーン切替
      │   ├─ キャッシュあり → 即時クロスフェード（ローディングなし）
      │   └─ キャッシュなし → ローディング表示 → ダウンロード → 表示
      │
      └─ ホットスポットクリック → 詳細パネル
          ├─ パネル即時表示（アニメーション付き）
          ├─ 製品名と画像は即座に表示
          ├─ 説明テキストは並列読み込み
          └─ 「読み込み中...」プレースホルダ → 実際の内容に置換
```
